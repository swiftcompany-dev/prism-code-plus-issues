# Prism Code Plus - Issue Tracker

**Languages:** [🇺🇸 English](README.md) | [🇰🇷 한국어](README.ko.md) | [🇯🇵 日本語](README.ja.md) | [🇨🇳 中文](README.zh.md)

**Prism Code Plus**の公開イシュートラッカーへようこそ!

このリポジトリは、Visual Studio Code用の自律型AIコーディングエージェントであるPrism Code Plusのバグレポート、機能リクエスト、コミュニティディスカッション専用です。

## 🤖 Prism Code Plusについて

Prism Code Plusは、自律的なAIコーディング機能をIDEに直接もたらす強力なVS Code拡張機能です。複数のAIプロバイダーをサポートしています:

- 🧠 **Anthropic Claude** (Sonnet 4.5, Opus 4, Haiku)
- 🔮 **Google Gemini** (2.5 Pro, 2.5 Flash)
- 💬 **OpenAI GPT** (GPT-4o, o1, o3-mini)
- 🌐 **OpenRouter** (200以上のモデルにアクセス)
- 🦙 **ローカルモデル** Ollama経由
- ...その他多数!

### 主な機能

✨ **自律コーディング** - 承認を得てファイルを作成、編集、コマンドを実行
🌍 **ブラウザ統合** - Webコンテンツをスクレイプし、ドキュメントを調査
🔧 **MCPサポート** - Model Context Protocolサーバーで拡張
⚡ **独自キー持参** - 独自のAPIキーまたは管理サービスを使用
🎯 **マルチモデルサポート** - プロバイダー間でシームレスに切り替え
📝 **タスク管理** - 組み込みタスク追跡と履歴

## 📝 イシューの提出方法

### 適切なテンプレートを選択

**4つの言語**でイシューテンプレートを提供しています: 英語、韓国語(한국어)、日本語、中国語(中文)。

イシューを作成する際は、**イシュータイプ**と**希望言語**の両方に一致するテンプレートを選択してください:

#### イシュータイプ

- 🐛 **バグレポート** - エラー、クラッシュ、予期しない動作
- ✨ **機能リクエスト** - 新機能または改善
- 📖 **ドキュメント** - ドキュメントのエラーまたは改善
- ⚡ **パフォーマンス** - 遅いレスポンス、高いリソース使用
- 🎨 **UI/UX** - インターフェースまたは使いやすさの改善

### クイックスタート

1. [Issues](../../issues)に移動
2. **New Issue**をクリック
3. テンプレートを選択 (例: "🐛 バグレポート (日本語)")
4. 詳細情報を記入
5. 提出!

## 📋 良いイシューの書き方

### バグレポートの場合

✅ **推奨:**
- バグレポートテンプレートを使用
- Prism Code Plusバージョンを含める
- 使用したAIプロバイダーとモデルを指定
- 明確な再現手順を提供
- 関連するエラーメッセージまたはスクリーンショットを含める

❌ **非推奨:**
- 機密情報を投稿 (APIキー、認証情報)
- 正当な理由なくテンプレートをスキップ
- 低性能モデルでのみ発生する問題を報告

### 機能リクエストの場合

✅ **推奨:**
- まず類似のリクエストを検索
- ユースケースとメリットを説明
- 望ましい動作を具体的に記述
- 実装の複雑さを考慮

❌ **非推奨:**
- AIプロバイダーの規約に違反する機能をリクエスト
- 既存のリクエストを重複

## ❓ FAQ

### 一般

**Q: Prism Code Plusはオープンソースですか?**
A: いいえ、Prism Code Plusはプロプライエタリプロジェクトです。このリポジトリは、イシューとコミュニティフィードバック専用の公開トラッカーです。

**Q: どのAIモデルでテストすべきですか?**
A: バグレポートは**Claude Sonnet 4.5**または同等のモデルで再現可能である必要があります。Prism Code Plusは複雑なプロンプトを使用するため、低性能モデルでは正しく動作しない可能性があります。

**Q: コードで貢献できますか?**
A: Prism Code Plusはプライベートリポジトリで開発されているため、直接のコード貢献は受け付けていません。ただし、イシューレポート、機能提案、ドキュメント改善を通じて貢献できます。

### イシューについて

**Q: イシューを提出しましたが応答がありません**
A: イシューは影響とリソースに基づいて優先順位が付けられます。重要なイシューは、ラベルとリアクション(👍)を通じて優先度が上がることがあります。緊急の場合は、[Discord](https://discord.gg/Pr9SuQJzkG)でお問い合わせください。

**Q: バグかどうかわかりません**
A: 不明な場合は、まず[GitHub Discussions](../../discussions)に投稿してください。コミュニティまたはチームがバグかどうか判断し、必要に応じてイシューに変換できます。

**Q: 重複したイシューを投稿してしまいました**
A: 問題ありません! チームが`duplicate`ラベルを付け、元のイシューに案内します。元のイシューに👍リアクションを追加して関心を示してください。

## 📊 イシューラベル

イシューの状態を追跡するためにラベルを使用します:

- 🏷️ **bug** - 確認済みバグ
- 🏷️ **enhancement** - 機能リクエスト
- 🏷️ **documentation** - ドキュメント改善
- 🏷️ **performance** - パフォーマンス問題
- 🏷️ **ui/ux** - UI/UX改善
- 🏷️ **good first issue** - 初心者向けの簡単な問題
- 🏷️ **help wanted** - コミュニティの意見が必要
- 🏷️ **wontfix** - 実装されない
- 🏷️ **duplicate** - 既に報告済み
- 🏷️ **investigating** - 調査中
- 🏷️ **in progress** - 作業中
- 🏷️ **needs more info** - 追加情報が必要
